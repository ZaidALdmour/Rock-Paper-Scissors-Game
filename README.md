# Rock-Paper-Scissors Game (C++)

A console-based Rock-Paper-Scissors game written in C++ using fundamental programming concepts such as Structures, Enums, and Functions.

## Features
* Custom Rounds: Allows the user to select between 1 to 10 rounds per game.
* Dynamic Screen Colors: Changes the console background color to Green on Player Win, Red on Computer Win, and Yellow on a Draw.
* Detailed Statistics: Displays comprehensive round summaries and an overall game results dashboard at the end.

## Requirements and Compilation
To run this game, you need a C++ compiler or an IDE like Visual Studio.

1. Open Rock Paper Scissors Game.cpp in your C++ IDE.
2. Compile and run the project.




# Rock-Paper-Scissors Game (C++)

A console-based Rock-Paper-Scissors game written in C++ using fundamental programming concepts such as Structures, Enums, and Functions.

## Features
* Custom Rounds: Allows the user to select between 1 to 10 rounds per game.
* Dynamic Screen Colors: Changes the console background color to Green on Player Win, Red on Computer Win, and Yellow on a Draw.
* Detailed Statistics: Displays comprehensive round summaries and an overall game results dashboard at the end.
* Clean Code Architecture: Built without complex templates, external STLs, or OOP classes to maintain core foundational logic.

## Equivalent Programming Problems in This Project

| Concept | Programming Problem Covered |
| :--- | :--- |
| Generating Random Numbers | Generate a random number within a range (RandomNumber()). |
| Handling User Input | Read and validate user input (Rock-Paper-Scissors choices). |
| Working with Enums | Define and use enum types for game choices and winners. |
| Conditional Logic | Implement decision-making using if and switch-case. |
| Using Structures (Structs) | Store structured game data (stRoundInfo, stGameResults). |
| String Manipulation | Convert enum values into readable strings (ChoiceName(), WinnerName()). |
| Looping (For & While Loops) | Iterate through multiple rounds using loops. |
| Creating Functions | Write reusable functions to handle various tasks (ReadPlayer1Choice(), PlayGame()). |
| Generating Computer Choices | Simulate AI behavior by randomly selecting a move. |
| Comparing Values | Determine winners based on user vs. computer choices. |
| Game Loop & Replay Feature | Allow the user to play multiple rounds with replay functionality. |
| Clearing the Screen (System Calls) | Use system("cls") to refresh the screen between games. |
| Tracking Scores | Store and update player wins, computer wins, and draws. |
| Working with Boolean Logic | Compare values to determine if the game should continue. |
| Reading and Returning Values from Functions | Use return values for WhoWonTheRound(), WhoWonTheGame(). |
| Switch-Case for Decision Making | Implement a structured approach to evaluating game logic. |
| Using srand(time(NULL)) for Unique Random Numbers | Ensure different results in each game. |
| Handling Edge Cases | Validate user input to prevent invalid choices. |
| Implementing a Simple AI (Computer Player) | Generate non-human opponent decisions. |
| Debugging & Output Formatting | Print structured results with clear formatting (PrintRoundResults()). |
